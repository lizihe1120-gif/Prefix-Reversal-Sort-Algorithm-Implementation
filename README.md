# 前缀反转排序算法实现
本项目实现了四种基于**前缀反转**（Prefix Reversal，也称“煎饼排序”）的排序算法，用于将不同特性的序列（随机排列、三调排列、二进制序列、三进制序列）排序为升序序列。算法结合了贪心策略、分治思想与有限深度搜索，并提供了完整的测试框架。

## 项目结构

## 算法简介

### 1. 通用排序（`pr_general_sort`）
- **目标**：对任意排列进行排序，尽量使用较少的前缀反转次数。
- **策略**：
  1. 先使用标准**煎饼排序**（每次将最大元素翻转到正确位置）获得基准序列。
  2. 对前 8 个元素使用优化的**暴力搜索**（`Violence` 函数），寻找更短的反转序列。
  3. 比较两种方案的总步数，选择较优者。

### 2. 三调序列排序（`pr_tritonic_sort`）
- **三调序列定义**：先增、后减、再增的排列（如 `[1,3,5,4,2,6,8]`）。
- **策略**：
  1. 找到第一个峰值（递减开始的位置）。
  2. 定位递减段中的最小值。
  3. 通过三次前缀反转将最小值移到正确位置。
  4. 剩余部分使用标准煎饼排序完成。

### 3. 二进制序列排序（`pr_binary_sort`）
- **目标**：将仅含 0 和 1 的序列排序为所有 0 在前、1 在后。
- **策略**：
  - 若最后一个元素为 0，先整体反转一次。
  - 扫描数组，每遇到 0→1 或 1→0 的边界，就进行一次前缀反转，将边界左侧部分倒序。
  - 算法保证在 O(n) 步内完成。

### 4. 三进制序列排序（`pr_ternary_sort`）
- **目标**：将含 0、1、2 的序列排序为所有 0、1、2 各自连续。
- **策略**：
  1. **第一阶段**：将所有的 2 移动到末尾（类似煎饼排序中处理最大值）。
  2. **第二阶段**：对剩余的前缀（只含 0 和 1）调用二进制排序算法。

## 辅助函数

- `reverse_prefix(arr, i)`：反转数组的前 `i+1` 个元素（原地修改）。
- `Violence(original_arr, target_arr, max_steps)`：使用广度优先搜索（BFS）在有限步数内寻找最优反转序列，通过状态剪枝避免循环。该函数仅用于通用排序中处理小规模子问题（前 8 个元素）。

## 测试与验证

### 运行测试
```bash
python pr_sorting_tester.py
```

测试脚本会：

生成指定类型和大小的测试数组（默认大小 50，每类 5 次测试）。

调用对应的排序算法，获得反转索引序列。

验证应用反转后数组是否有序。

统计总反转次数，若任一测试失败则返回 -1。

测试输出示例
text
Testing pr_general_sort
    testing array: [23, 5, 12, ...]
    reversals: [2, 5, 3, ...]
        test passed!
        num reversals: 42
...
total number of reversals: 315
项目亮点
混合策略：通用排序中结合标准煎饼排序与局部暴力搜索，在保证可解性的同时尝试优化步数。

状态剪枝：暴力搜索函数通过访问集避免重复状态，并对搜索深度进行限制。

算法模块化：二进制排序作为子程序被三进制排序复用，代码结构清晰。

详尽报告：PDF 报告中记录了算法设计的思考过程、遇到的挑战（如断点启发式尝试与放弃）及最终方案。
